# Contributing

## Branch Naming
Use short, descriptive branch names.

Examples:
- `feat/doc-intelligence`
- `fix/health-response`
- `docs/project-structure`

## Commit Style
Use concise imperative commit messages.

Examples:
- `docs: update project structure`
- `refactor: centralize constants`
- `chore: remove placeholder files`

## Folder Conventions
- Keep each folder focused on one responsibility.
- Prefer feature-oriented grouping.
- Avoid introducing duplicate placeholder structures.
- Add new modules only when they have a clear purpose.

## Code Style
- Preserve existing architecture.
- Avoid changing behavior when editing branding or documentation.
- Use explicit names for shared constants.
- Keep implementation files small and focused.

## Documentation Rules
- Document new folders when they are added.
- Keep architecture and roadmap docs aligned.
- Update branding and terminology consistently.
- Prefer concise, factual descriptions.
